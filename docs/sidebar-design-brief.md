# GlanceTerm 侧边栏 —— 设计需求 Brief

> 给设计师的交接文档。目标：重新设计 GlanceTerm 的「AI Tabs」侧边栏，使它在高信息密度下仍然**一眼可扫视**，状态清晰、层级分明、视觉平静不刺眼。

---

## 1. 产品背景

**GlanceTerm** 是一个面向开发者的终端 App（基于 Tabby 的分支，Electron + Angular，深色主题为主）。它的核心使用场景：开发者**同时开很多个终端 tab，每个 tab 里跑一个 AI 编码 agent**（Claude Code / Codex / Gemini / opencode）。

问题：tab 一多，用户就不知道哪个 agent 在干活、哪个跑完了、哪个正卡着等你确认权限。**侧边栏就是解决这个的「总览仪表盘」**——把所有 tab 的 AI 状态聚合到一个竖直面板里，让用户**扫一眼就知道该去处理哪个**。

类比：像 Slack 的频道未读列表、或 CI 的 job 列表——核心价值是「在一屏内、用最低认知成本，看清一堆并发任务各自的状态」。

---

## 2. 核心用户场景（Job-to-be-done）

1. **我现在该处理哪个？** —— 哪个 agent 在等我确认权限 / 跑完了等我看结果。这是**最高优先级**，要最显眼。
2. **哪些还在忙？** —— 不用管，让它跑。
3. **哪些空闲/已结束？** —— 次要信息。
4. **点一下跳到那个 tab。** —— 列表项即导航。
5. **顺手操作** —— 截图丢给当前 agent、给当前 tab 开一个并排 shell、调设置。

---

## 3. 侧边栏主要功能（信息架构）

侧边栏从上到下分 5 个区域：

### 3.1 顶部标题区（Header）
- 当前只有一个标题文字「AI Tabs」。可重新设计（含品牌感、可放聚合状态）。

### 3.2 过滤器（Filter Pills）
- 一排可点的筛选标签，每个带数量徽标：
  - **All（全部）** · **Needs You（需要你）** · **Done（已完成）** · **Working（进行中）** · **Idle（空闲）**
- 当前选中态高亮。数量为 0 的筛选项仍显示。

### 3.3 Agent 列表（核心，重点设计）
每个 tab 是一行（row）。一行最多承载这些信息（很多是条件显示）：

| 元素 | 说明 | 是否常驻 |
|---|---|---|
| 序号 | tab 在顶部 tab 栏的序号（1、2、3…） | 常驻 |
| 状态灯（dot） | 左侧竖条上的圆点，颜色=状态；working 时**脉冲呼吸** | 常驻 |
| 置顶标记（pin） | 被置顶的行显示一个图钉图标 | 条件 |
| 主标题 | 工作目录名（如 `glanceterm`）；没有 cwd 时用 tab 标题 | 常驻 |
| 状态文字 | working / needs you / done / ready / shell | 常驻 |
| Agent + Model 标签 | 一个底色小药丸：`Claude  opus-4-8[1m]`（工具名 + 模型名） | 仅 AI tab |
| Token 用量 | `27k input  284k output`（本会话累计） | 仅 AI tab |
| 并发计数 | `2 agents · 3 shell · 9 monitor`（子 agent / 后台任务 / 监控任务数） | 条件（>0 时） |
| 完整路径 | `~/work/glanceterm`（截断的完整 cwd） | 条件（有 cwd） |
| 最近活跃时长 | 右侧的 `3m`、`12s`（距上次活动多久） | 条件 |

行的交互态：
- **hover**
- **active**（当前正聚焦的 tab，要明显区分）
- **pinned**（置顶）
- **subordinate**（从属，例如 split 里的次要面板，视觉上要弱化）

### 3.4 状态体系（Status Taxonomy）—— 设计核心难点

状态决定了状态灯、状态文字、整行强调程度。**5 种状态**：

| 状态 | 文字 | 当前色 | 语义 | 视觉优先级 |
|---|---|---|---|---|
| Needs Permission | `needs you` | 橙 `#FFAA55`（品牌色）| agent 卡住等你确认（最该处理） | **最高**，要抢眼 |
| Done | `done` | 红 `#FF5252` | 跑完一轮、你还没看 | 高 |
| Working | `working` | 绿 `#4CAF50`（脉冲）| 正在干活 | 中 |
| Idle / Ready | `ready` | 蓝 `#5B9EF5` | 空闲等输入 | 低 |
| No AI | `shell` | 灰 `#6B7178` | 普通 shell，没跑 agent | 最低，几乎隐身 |

> ⚠️ **重要教训**：状态色彼此**不能混淆**。之前 Claude 的 agent 标签用了粉红 `#E879A6`，结果和 done 的红、以及状态语义撞色，用户误以为是「状态」。**Agent/Model 标签是元数据，不是状态，绝不能用任何接近状态色的颜色**——这是这次重设计要解决的痛点之一。

### 3.5 多 Agent 支持
支持 4 种工具，需要在视觉上可区分（但**不能用状态色**）：
- **Claude**（主力，最常用）· **Codex** · **Gemini** · **opencode**
- 当前做法：每个工具一个色相的小药丸标签。新方案可以保留弱色相区分，或统一中性、用文字/图标区分——由设计师权衡。

### 3.6 排序与置顶逻辑（行为，需视觉表达）
- 排序优先级：**needs you > done > 其余按 tab 栏顺序**。
- 「done → 你点开看过 → 变 idle」会有一个 ~2 秒的「短暂置顶」过渡，避免列表在你眼皮底下跳动。
- 设计需要考虑：**列表重排时的过渡动效**要平滑、不闪。

### 3.7 底部统计（Footer）
- 一行聚合数字：`2 done · 5 working · 3 idle · 1 need you`。

### 3.8 底部工具栏（Action Toolbar）
3 个按钮：
1. **截图**（主按钮 + 一个下拉箭头给「截图时隐藏窗口」选项）—— 截屏并粘贴给当前聚焦的 agent。
2. **并排 Shell** —— 给当前 tab 开一个 split shell。
3. **设置**（齿轮）—— 打开设置弹窗。
- 按钮有「当前 tab 不是 AI」时的弱化态、截图中的 busy 态。

### 3.9 空状态（Empty States）
- 完全没有 AI agent 时：插画 + 「Nothing to glance at yet」引导文案。
- 筛选后为空 / 全部被隐藏时：对应的空状态文案。

### 3.10 设置弹窗（Settings Modal）
- 居中模态，每项设置含：标题 + 说明 + 开关。当前 3 项：
  - 完成时提示音（Chime on agent done）
  - 自动批准权限（Auto-approve，默认关）
  - 隐藏没有 agent 的 tab

### 3.11 规划中（Groups 视图，可一并出概念稿）
- 按 git 仓库根目录**自动分组**，提供「列表 / 分组」视图切换。希望设计师能给出分组视图的概念方向。

## 4. 设计需求与约束

### 4.1 形态与尺寸
- **终端 App 内嵌的竖直侧边栏**，可由用户拖拽改变宽度。
- 设计基准宽度 **≈360px（最小 360）**，需考虑更宽时的表现。高度撑满窗口、内容区可滚动。
- 实现为 Web/CSS（Angular 组件），**避免依赖原生独占效果**；动效用 CSS 可实现的为佳。

### 4.2 视觉气质
- **深色主题为主**（必做），如有余力可给浅色版本方向。
- 开发者工具调性：克制、专业、信息密度高但不杂乱。**平静**，不要花哨刺眼（用户明确反馈过「不要太抢眼的颜色」）。
- 品牌主色：蜂蜜橙 `--gt-accent: #FFAA55`（也是「需要你」的状态色）。

### 4.3 核心设计目标（按重要性）
1. **Glanceable 可扫视**：一屏尽量多行，用户扫一眼就能定位「需要我处理的」。
2. **状态层级分明**：needs you / done 必须比 working / idle 更突出；shell 几乎隐身。
3. **状态色互不混淆**（见 3.4 教训），并保证色盲可辨（不能只靠颜色，配合文字/形状）。
4. **元数据安静**：agent 名、模型、token、路径这类元数据用中性弱色，不与状态争夺注意力。
5. **重排平滑**：状态变化导致的排序变动要有平滑过渡。

### 4.4 现有设计 Token（供延用或重构）
```
品牌:   --gt-accent #FFAA55   --gt-accent-deep #FF7A3D   --gt-accent-soft rgba(255,170,85,.14)
文字:   --gt-text #E7E9EC     --gt-text-dim #9BA1A9      --gt-text-faint #6B7178
描边:   --gt-border rgba(255,255,255,.07)
状态:   working #4CAF50   ready/idle #5B9EF5   done #FF5252   need-you #FFAA55   shell/none #6B7178
字体:   路径/模型/token 用等宽（mono）；标题可用系统 UI 字体
```

### 4.5 可访问性
- 文本对比度满足 WCAG AA；状态不能仅靠颜色传达。
- 交互元素有清晰 focus / hover 态。

## 5. 期望交付物（Deliverables）

请设计师产出：
1. **整体侧边栏布局**：有内容（多行混合状态）+ 空状态，至少深色版。
2. **单行精细稿（row）**：覆盖全部 5 种状态 × 关键变体（active / pinned / 含并发计数 / 纯 shell）。
3. **状态色板 / 设计 token**：明确每种状态、每个工具、各类文字层级的颜色，并说明如何避免互相混淆。
4. **过滤器、底部统计、工具栏、设置弹窗**的样式。
5. **Groups 分组视图**的概念方向（可粗）。
6. **交互态规范**：hover / active / pinned / subordinate / 列表重排过渡。
7. **标注**：间距、字号层级、圆角、颜色 token，方便前端还原。
